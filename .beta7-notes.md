## CRITICAL SCOPE FIX

### The Real Issue Found!
When calling async_set_unique_id() in async_step_user(), we were using:
- Wrong: `await self.async_set_unique_id(DOMAIN)` (imported variable)
- Correct: `await self.async_set_unique_id(self.DOMAIN)` (class attribute)

This caused Home Assistant to fail when creating the config flow handler.

### What's Fixed
Now uses the class attribute self.DOMAIN which Home Assistant expects for proper handler registration.
